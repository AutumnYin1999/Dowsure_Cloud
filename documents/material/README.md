# material —— 服务商 Deal Agent 的"可信度"参考材料库

> 用途：给豆服云**服务商诊断/成交 Agent** 的每一句"数字化"话术提供**可追溯的出处**。
> Agent 报的任何金额、比例、系数，都应能在本目录里找到来源 + 信心等级，而不是 AI 现编。
> 建立日期：2026-06-09 ｜ 维护：随研究持续追加。

## 范围（本轮）
- 锚定画像：**北美海外仓 / 头程物流服务商**（中国→美国跨境主干道；美东/美西仓）。
- 覆盖系数：账期与回款、坏账率/DSO、融资额度倍数、B2B 线索转化率、物流成本占营收比。

## 信心图例（沿用代码里 `sellerEconomics.ts` 的习惯）
- 🟢 **硬**：多源一致，或权威报告/官方产品文件给出明确数字。
- 🟡 **中**：单一来源、区间反推，或用通用行业基准代理。
- 🔴 **假设**：无直接背书，**待豆沙包内部数据校准**——Agent 用到时必须标"估算/区间"。

## ⚠️ 一条必须诚实的边界
本目录的外部数字是**公开行业基准（代理值）**，不是豆沙包自有客群的真实均值。
- 公开基准足够支撑**可信的第一版**（且每条有出处）。
- 但真正的护城河系数——海外仓客群的**真实平均账期、坏账率、TermPay 放款额分布**——只在豆沙包 3PAR/TermPay 后台。拿到后应**优先替换**本表中标 🔴/🟡 的代理值。
- 已知的豆沙包**自有一手数据**（24 份问卷等）也收录在系数表里，并明确区分"内部 vs 外部"。

## 红线（与 seller 引擎一致）
> 数字只在确定性引擎里算（单一数据源、可追溯到本目录），**AI 永远不碰数字、只把引擎算好的数讲成话**。运行时不让 Agent 现搜现编。
