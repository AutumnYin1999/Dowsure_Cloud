## Agent 2 · 账期「融资路径对比」内联模块（待执行）

# 任务：在卖家账期诊断结果中插入「三种筹资方式对比 → TermPay」（Agent 2）

## 项目
Dowsure_Cloud（React 18 + Vite + TS + Tailwind，深色 Agent OS 风格，纯前端 mock）。
改动文件：src/components/SellerHomePage.tsx（卖家智能服务台，账期测算在 computeTerm / 账期结果区）。
本轮只做前端展示，不接后端、不做真实申请。

## 背景与位置
卖家账期链路：账单诊断 → 判断现金是否够付 → （不够时）该怎么筹钱 → 落到 TermPay。
现状：computeTerm 算出压力等级（低/中/高/待确认）后，结果区直接给「TermPay 预申请」按钮，
中间缺少「为什么是 TermPay 而不是贷款/股权」的对比，故事是断的。

要在【账期判断结果卡之后、TermPay 预申请按钮之前】内联一个「三种筹资方式对比」模块。

## 触发规则（按压力等级分级）
- 压力「高」（现金不够）：完整展示三栏对比，金融科技栏高亮，底部接 TermPay 预申请。
- 压力「中」：展示轻量入口（一行「想了解其他筹资方式？」可展开看对比），不强推。
- 压力「低」：不展示（不缺钱，避免变推销）。
- 「待确认」：可复用「中」的轻量入口。

## 三栏内容（按用户既定口径，纯客观对比，不夸大）
1. 银行贷款：申请周期长、审核严格、需到期还本付息（可能需抵押/担保）。
2. 股权融资：资金确定性低、尽调资料复杂、周期久、稀释股权且需让渡部分经营参与。
3. 金融科技产品（TermPay，豆分期升级版）：申请简单、批复快、随借随还、按账期灵活延期/分期。
   → 自然作为最契合「大额账单+回款错配」场景的选项胜出，但不要用"最佳/保证/独家"等夸张词。

## UI 要求
- 内联在账期结果区，沿用现有 seller-desk.css 暗色卡片风格，三栏在桌面横排、移动端纵排。
- 金融科技栏用现有高亮/渐变描边样式区分，但保持克制。
- 模块底部承接现有「TermPay 预申请」CTA（openContact('TermPay 预申请')）。
- 加一行合规小字：额度/费率/期限以合作资金方审核与正式协议为准，Dowsure 不放款。

## 约束
- 纯前端 mock；复用现有 computeTerm 的等级结果，不改其计算口径。
- 不新增路由/页面（内联即可）；不引入新依赖。
- npm run typecheck 必须通过。

## 验收
- 账期压力「高」：结果卡下方出现三栏对比，金融科技栏高亮，可点 TermPay 预申请。
- 「中/待确认」：出现可展开的轻量入口，展开后见同一份对比。
- 「低」：不出现对比模块。
