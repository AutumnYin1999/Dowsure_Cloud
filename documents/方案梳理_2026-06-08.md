## 背景与定位

- **豆服云**：当前要研究的产品（服务商增长引擎，权益包 + AI 拓客 + 金融赋能）。
- **豆分期 (DPL)**：另一个产品（"先用后付"信用支付工具），目标是**把它和豆服云链接起来**。
- **Dowsure / Agent OS / 6 Agent / Series C**：CEO 的战略规划，是**终极目标**，不是当前落地范围。
- **对话式 agent 形式**：CTO 已做的 benchmark（`aiceo.dowsure.com` 的 AI 高管团队），CEO 与本人都认可这种表现形式；项目里已有雏形（上次试做的 `/seller/chat` 卖家服务台）。

## 结论一：靠"问问题"引导用户，可行 —— 但问题要"倒着设计"

### 1.1 为什么对话式引导能成立（4 个心理杠杆）
1. **降负荷**：一次只问一个问题，远比一张大表轻（解决卖家侧"问得太多"的反面）。
2. **每答一题给一口糖**：用户每回答一次，agent 立刻回一个数字/洞察。
3. **承诺逐步升级 (foot-in-the-door)**：先答无痛小问题→拿到价值→产生投入感→再要手机号/授权就不抗拒。
4. **路径收敛**：用户以为在自由聊天，其实每个答案都在一棵**很小的决策树**上走向某个叶子，**而每个叶子都是我们真有的东西**。

> ⚠️ 关键约束：只能引导到"我们真有的结局"。所以问题不是从"想问什么"出发，而是**从结局倒推**。

### 1.2 问题设计：倒着设计三步法
1. **先列结局**（想把人导到哪）
   - 卖家线 → `找服务商` / `账期·TermPay(豆分期)` / `两者都要`
   - 服务商线 → `获客线索` / `账期风控` / `品牌曝光`
2. **找出能同时①分流②算钱的最小问题集**（一题两用）。
3. **把"最大的那个金额"放在要手机号之前**：小问题→小洞察→…→砸一个最大钱数→紧接 CTA。

### 1.3 卖家侧 3 问（已验证够用）
| 问题 | ①路由作用 | ②算钱作用 |
|---|---|---|
| 主要哪个平台？（兼身份分流） | 行业基准/合规/相关服务商 | 不同平台回款周期≠→占压系数 |
| 月 GMV 区间？ | 卖家分层 L1–L4 | **金额的乘数**（账压、缺货损失按比例算） |
| 现在最头疼啥？ | 路由到 找服务商/账期/都要 | 决定主推哪个金额 |

### 1.4 "算钱"示例（透明 mock，可落地）
卖家：Amazon、月 GMV 200 万、最痛=资金紧 →
- 「Amazon 回款+结算在途，约 **11–12%** 销售额压在回款里」→ **¥23.7 万现金被压住**（截图实例正是 11.85%，系数真实）
- 「旺季高动销 SKU 断货 3 天 ≈ **¥18 万缺货损失**」
- 「这笔海外仓账单可走豆分期延期 90 天 → 预计释放现金流 ¥X」→ **[生成 TermPay 预申请]**

> 每个金额 = `GMV × 写死的系数`，函数单列，后续接后端只换这一处。这就是把"问卷推荐"升级成"AI 成交顾问"的关键动作。

### 1.5 从 benchmark 偷的 UI：顶部"价值感状态栏"
benchmark 顶部一条状态栏，随对话**逐格填满**：
- 卖家线：`访客 → 账压金额 → TermPay 可释放现金 → 预申请`
- 服务商线：`访客 → 线索数 → 预计成交 → 推荐版本`

用户眼睁睁看着自己"从访客变成有额度/有方案的人"——这就是 CEO 说的"先制造价值感"的可视化载体。

> 开场第一问用 **JTBD 的"任务"语言**而非"属性"语言：别问"你的身份是？"，问"你现在最想搞定哪件事？"

## 结论二：豆分期对标 Payoneer Capital Advance —— 产品逻辑站得住，且有可抄之处

### 2.1 对标确认（三家几乎同一个模子，定价形成行业共识）
| | 额度公式 | 获客方式 | 还款 | 费用 |
|---|---|---|---|---|
| **豆分期 DPL** | GMV **×1.5**，最高 $100 万 | —（嵌在 Dowalet） | T+90 自动扣 | 免息(服务商贴息)/固定手续费 |
| **Payoneer Capital Advance** | 月流水 **×1.4**，封顶 $75 万 | 邀请制（系统主动给 offer） | 未来收入按比例扣 | 一次性固定费、事先透明 |
| **Wayflyer** | 月营收 **×1.5–3**，额度 $5k–$2000万 | 自助申请（连只读数据） | 按经营周期还本金 | 一次性固定费、无隐藏 |

**两个洞察：**
1. **额度系数行业共识 ≈ 月营收的 1.4–3 倍**，豆分期 ×1.5 落在区间下沿——偏保守、安全，对外讲完全站得住（可当可信度背书）。
2. **获客两条路**：Payoneer 邀请制（高信任但被动）vs Wayflyer 自助申请（主动但要先授权数据）。**我们的对话式 agent 正好取两者之长**——像 Wayflyer 让用户自己进来聊，聊完像 Payoneer "主动弹一个额度给你领"，而非填申请表。

> 结论：豆分期的产品逻辑不是闭门造车，海外头部支付公司(Payoneer)+ 营收型融资(Wayflyer)用的是同一套打法。

### 2.2 可直接抄的 3 件事
1. **"领取"而非"申请"**：Payoneer 是 invitation-only，系统看流水主动给 offer，没有被拒的尴尬。
   → 我们 agent 聊完算出额度后，**直接弹"你已获得预估额度 ¥X"**，而不是丢一个申请表。契合"先制造价值感"。
2. **额度公式可视化**："月流水×140%"这种一句话能讲清的公式，比黑箱评分更让人信。
   → agent 算额度时把公式亮出来："按你 GMV 的 1.5 倍，你大概能拿 ¥X。"
3. **"签前就知道总成本"**：固定费、透明，是它反复强调的信任锚点。
   → TermPay 预申请页照搬"无隐藏费用"话术。
4. **分步骤标耗时**（Wayflyer：填信息 2 分钟 / 授权连数据 10 分钟 / 24 小时放款）。
   → 用在我们对话流程，缓解"要填多久"的焦虑。
5. **额度公式行业共识（×1.4–3）**可作为豆分期 ×1.5 对外的可信度背书。

### 2.3 行业背书（来自报告，可当 agent 开场钩子）
- **现金流卡在"回款时间差"是卖家第一痛点**：JungleScout 报告原话——卖家付钱(备货/广告)与亚马逊回款之间的 payment timing gap 不断放大现金缺口。这正是豆分期/TermPay 存在的全部理由。
- 行业数字：**71% 企业"增收不增利"**；**超六成亚马逊中小卖家利润下滑，31% 降幅超 50%**；FBA 备货+头程+仓储预付是现金流压力主因。
- 卖家头号痛点排名：海运费涨 **38%** / 货物成本涨 **34%** / 广告费涨 **32%** / 怕被三方卖家抢份额 **~1/3**。

## 结论三：亿邦《2025 全球跨境电商供应链报告》—— 给"算钱"系数和服务商线主张提供背书

> 来源：亚马逊全球开店 + 对外经贸大国际商务研究中心 + 亿邦智库（2025-08）。核心调研 308 份商家问卷（2025-06）。物流/供应链视角，一箭双雕：既喂卖家线算钱系数，又是服务商线（物流服务商=豆服云核心客户）的认知地图。

### 3.1 给"算钱"的硬系数（最值钱）
1. **物流仓储成本占营收 15–20%**（商家财报抽样）。成本结构：产品 20-30% / **物流仓储 15-20%** / 广告营销 10-15% / 销售佣金及服务费 8-15%。
   → 月 GMV 200 万 → 物流仓储支出 ≈ **30–40 万/月**。豆分期释放的现金压力、服务商账期盘子按此算。
2. **旺季缺货损失 ≈ 销售额的 7–14%**（从亚马逊 AWD 案例反推：自动补货使"缺货率降 15%、销售额提升 7%–14%"）。
   → 缺货损失系数改为 `月GMV × 7~14% × 断货时长占比`，替代此前拍脑袋的数。
3. 库存痛点：滞销积压 **61.2%** / 旺季缺货丢损 **33.9%** / 海外仓"仓储成本过高" **55.1%**。新品开发周期约 **170 天**（普华永道）。

### 3.2 服务商选择物流服务的 Top5（= 豆服云权益包卖点排序）
| 因素 | 占比 |
|---|---|
| **平台流量扶持 / 提升新品曝光销量** | **69.1%** |
| 行业口碑 | 58.0% |
| 全链路一体化服务能力 | 47.2% |
| 稳定性与可靠性 | 37.8% |
| 价格 | 31.9% |
| AI 大数据工具支持（如库存预测） | 22.5% |

### 3.3 三个直接命中豆服云/豆分期的痛点
- **71%** 苦于"多服务商分段对接、信息孤岛、货损责任难界定" → **57% 亟需一站式端到端方案**。→ 豆服云"嵌入式金融 + 一站式"接得上。
- **52.8%** 被"入仓费用不透明/隐藏费用"困扰。→ 豆分期"费率透明、无隐藏费用" + "账期优化风控模型"的切入口。
- 能力缺口：品牌营销 **67%** / 生产库存管理 **63%** / 本土化 **62.5%** / 产品研发 59%。→ 豆服云"AI 品牌营销赋能""AI 拓客"命中前两位。

### 3.4 怎么用进设计
- **卖家线算钱**：物流仓储 15-20% + 缺货损失 7-14%，两个系数均有报告背书。
- **服务商线主张**：开场锚定 **"流量扶持/获客"（69.1% 第一需求）**=豆服云"AI 拓客"核心；账期/费用不透明（52.8%）=挂载豆分期的切入点。

## 结论四：网经社《2025 跨境营销服务商观察》—— 服务商分类地图（数据偏薄，但界定客户有用）

> 诚实标注：这份偏"分类综述"，**未给出**服务商获客难/获客成本/坏账的量化数据。价值在分类。

### 4.1 服务商七大分类
1. B2C 平台型（Amazon、AliExpress、TEMU、TikTok Shop）
2. B2B 撮合型（Alibaba.com、Made-in-China、DHgate）
3. B2B 垂直型（海智在线等）
4. **技术服务型**（ShopLine 等建站 SaaS）
5. 软件工具型（ERP、BI、AI 客服）
6. **物流与海外仓型**（大健云仓等）
7. **整合营销型**（飞书深诺、卧兔网络等）

### 4.2 对豆服云的用处
- 豆服云服务商客户主要落在第 **6（物流海外仓）、7（整合营销）、4（技术服务）** 类。
- **豆分期天然服务第 6 类**：物流商/海外仓账期场景（物流费、仓储费延期）=豆分期核心场景。
- **AI 拓客/品牌营销服务第 7、4 类**：整合营销/技术服务商最需要获客线索。
- → **服务商线开场分流**可照此问："你是做物流/海外仓、营销、还是工具的？"不同类型挂载不同权益（物流→豆分期账期；营销→AI 拓客）。

### 4.3 数据缺口的应对
服务商"获客痛点"论据改用更扎实的内部材料：
- 豆服云自有 **24 份问卷**：获客难居首 **43.48%**、融资需求 **67%**、78.26% 期望年度套餐 ≥15 万。
- 亿邦报告：**71%** 多服务商分段 / **57%** 要一站式 / **52.8%** 费用不透明。
- 结论：**服务商线核心论据，内部一手数据 > 外部白皮书**。
